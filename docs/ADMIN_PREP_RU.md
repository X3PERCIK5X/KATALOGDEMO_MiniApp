# Подготовка Mini App к админ-панели

Документ фиксирует, какие части интерфейса уже переведены на конфигурацию и как их редактировать без правки кода.

## 1) Редактируемые блоки (контент)
- Главные баннеры: `config.json -> homeBanners[]`
- Слайды статей на главной: `config.json -> homeArticles[]`
- Статья «О продукте»: `config.json -> aboutText`
- Статья «Оплата и условия»: `config.json -> paymentText`
- Статья «Как работает»: `config.json -> productionText`
- Контакты: `config.json -> contactsText`
- Локации магазинов: `config.json -> storeLocations[]`

## 2) Каталог и карточка товара
- Категории: `data/categories.json`
- Товары: `data/products.json`

Ключевые поля товара для админки:
- `id` — уникальный id
- `categoryId` — привязка к категории
- `title` — название
- `shortDescription` — короткий текст в списках
- `description` — описание в карточке товара
- `specs[]` — характеристики
- `images[]` — галерея
- `price` / `oldPrice` / `discountPercent` — цена и скидка
- `badge` / `tags[]` — промо-метки
- `sku` — артикул

## 3) Что уже сделано для админки
- Главная страница рендерится из `config.json` (баннеры + статьи)
- Есть fallback-значения, если конфиг временно недоступен
- Контентные блоки имеют стабильные ids (`homeBanners[].id`, `homeArticles[].id`)
- Разделы привязаны к `data-screen` и открываются без хардкода в HTML

## 4) Что дальше добавить в админке
- CRUD для `homeBanners`, `homeArticles`, `storeLocations`
- CRUD для `categories`, `products`
- Валидация полей (длина, обязательность, тип)
- Предпросмотр темы (dark/light) перед публикацией
- Публикация изменений в JSON и обновление версии статики

## 5) Принцип публикации
- После сохранения админкой нужно менять версию ресурсов в `index.html`:
  - `style.css?v=...`
  - `app.js?v=...`
- Это гарантирует обновление в Telegram WebView без старого кеша.
